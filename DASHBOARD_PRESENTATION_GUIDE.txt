1. EXECUTIVE SUMMARY
================================================================================

Frammer AI Analytics Dashboard is a 6-page, fully browser-based, modular analytics
platform built in Next.js 15 (App Router) with TypeScript. It connects to a live
PostgreSQL database using a star schema data model.

The dashboard serves multiple personas:
  - Founders/Leadership   → Page 1 (Executive view)
  - Client Success        → Page 3 (Channel/client drill-downs)
  - Operations Teams      → Page 4 (Platform efficiency, velocity)
  - Editorial/Production  → Page 4 (Content flow, output mix)
  - Tech/Product          → Page 5 (Data explorer, raw tables)
  - Any stakeholder       → AI Chat (natural language queries)
  - Power users           → Page 6 (Custom widget builder)

Key differentiators over the baseline dashboard:
  - Multi-level drill-downs (L1 → L2 → L3 → raw records)
  - AI-powered natural language querying (Gemini + LangChain + RAG)
  - Prompt-driven custom KPI/chart builder (Page 6)
  - Inline insight definitions on every widget
  - Data health monitoring
  - Fully modular — new metrics/dimensions require zero redesign

================================================================================
2. ARCHITECTURE OVERVIEW
================================================================================

Frontend:
  - Next.js 15 App Router (React, TypeScript)
  - Tailwind CSS for styling
  - Recharts (bar, line, area, scatter, sankey, heatmap)
  - Chart.js via react-chartjs-2 (line, doughnut charts on pages 1 & 2)

Backend / API:
  - Next.js API Routes (app/api/**)
  - PostgreSQL database (via pg pool)
  - 10+ dedicated API routes per page
  - /api/drilldown  → raw video records with multi-filter support
  - /api/insights   → widget-level insight generation
  - /api/chat       → AI conversational SQL queries
  - /api/page6/*    → custom widget generation & persistence

AI / LLM Layer:
  - Google Generative AI SDK (@google/generative-ai)
  - LangChain + ChatGoogleGenerativeAI (Gemini models)
  - RAG (Retrieval Augmented Generation) for semantic context
  - Model: gemini-2.5-flash for widget builder; gemini-3-flash for chat agent

Data Model (Star Schema):
  Fact Table: videos
    - video_id, client_id, channel_id, user_id
    - input_type_name, output_type_name, language_name
    - published_platform, published_flag, published_url
    - uploaded_at, processed_at, published_at
    - duration (interval/numeric), headline

  Summary/Aggregate Tables:
    - monthly_processing_summary (month × client: uploaded, processed, published counts)
    - monthly_duration_summary (month × client: uploaded, created, published duration)
    - channel_processing_summary (channel × client: counts + durations + publish rate)
    - channel_user_processing_summary (channel × user: counts)
    - channel_wise_publishing_counts (channel: published count)
    - channel_wise_publishing_duration (channel: published duration)
    - input_type_processing_summary (input_type: counts)
    - output_type_processing_summary (output_type: counts)
    - language_processing_summary (language: counts)
    - user_processing_summary (user: counts)
    - custom_widgets (id, title, prompt, sql_query, widget_type, config, created_at)

3.1 KPI CARDS GRID (9 Business-Intelligence KPIs)
──────────────────────────────────────────────────────────────────────────────

Each card shows:
  - Primary value (latest month)
  - All-time total (secondary line)
  - MoM change percentage + direction arrow
  - "?" definition button (popover)
  - "Insight" button (loads AI-generated insight)

KPI 1: Est. Human Hours Saved
  Formula : SUM(total_uploaded_duration) × 3
  Unit    : Hours
  Logic   : 1h raw content → 3h manual editing equivalent eliminated
  Context : Measures AI ROI / value delivered

KPI 2: Time-to-Market / Production Velocity
  Formula : AVG(published_at − uploaded_at)
  Unit    : Hours
  Logic   : Lower = faster production pipeline
  Context : Key SLA metric for editorial teams

KPI 3: Content Waste Index
  Formula : total_created_duration − total_published_duration
  Unit    : Hours
  Logic   : AI-generated content that was NEVER published
  Context : Identifies wasted processing cost; should trend down

KPI 4: Client Concentration Risk
  Formula : (top_client_duration / total_duration) × 100
  Unit    : %
  Logic   : If one client = 60%+, that is a revenue concentration risk
  Context : Business health / diversity metric

KPI 5: Total Uploaded Volume
  Formula : COUNT(videos WHERE uploaded_at IS NOT NULL), SUM(duration)
  Unit    : Videos (count) + Hours
  Logic   : Total raw content entering the pipeline

KPI 6: Total AI-Generated Output
  Formula : COUNT(videos WHERE processed_at IS NOT NULL), SUM(duration)
  Unit    : Videos (count) + Hours
  Logic   : Total AI-processed outputs across all clients

KPI 7: AI Content Multiplier
  Formula : total_created / total_uploaded
  Unit    : Multiple (e.g., "8x")
  Logic   : For every 1 upload, how many AI outputs are generated
  Context : Measures AI amplification power

KPI 8: Period-over-Period Growth (Δ%)
  Formula : (current_month_combined − prev_month_combined) / prev_month_combined × 100
  Unit    : %
  Logic   : MoM growth across uploaded + published volume

KPI 9: Top Performing Output Type
  Formula : output_type with MAX(published count) in current month
  Unit    : Label + count
  Logic   : Which content format is driving the most publishing activity

──────────────────────────────────────────────────────────────────────────────
8.1 WIDGET GENERATOR
──────────────────────────────────────────────────────────────────────────────

Input: Natural language prompt (textarea)
Keyboard shortcut: Cmd/Ctrl + Enter to generate

Examples:
  "Total uploaded videos per client as a bar chart"
  "Monthly publishing trend this year as a line chart"
  "What is the total processed video count? (KPI)"
  "Top 5 languages by processed count as a pie chart"
  "Show channel vs publish rate as a table"

AI Pipeline:
  1. Fetch full DB schema (excluding clients + custom_widgets tables)
  2. Call Gemini 2.5 Flash with system prompt + schema + user prompt
  3. LLM returns JSON: { title, widget_type, sql }
  4. Validate SQL (no forbidden DDL/DML operations)
  5. Execute SQL against PostgreSQL
  6. Build chart spec (labels, datasets)
  7. Return preview data to frontend

LLM Safety:
  - Forbidden: DROP / DELETE / UPDATE / INSERT / ALTER / TRUNCATE / CREATE / GRANT / REVOKE
  - Only SELECT queries allowed
  - Returns error if unsafe SQL generated

Widget Types Supported:
  - kpi    → Single big number (one row, one aggregate column)
  - bar    → Grouped/stacked bar chart (Recharts BarChart)
  - line   → Line/trend chart (Recharts LineChart)
  - pie    → Donut/pie chart (Recharts PieChart)
  - table  → Scrollable HTML table
